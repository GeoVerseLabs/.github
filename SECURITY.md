# Security Policy

## Supported Versions

Our open-source projects are young and actively developed on a single track. Security fixes
are applied to the latest released version on `main` / the latest published package version;
older versions are not separately patched unless stated otherwise in a repo's own docs.

| Repo | Supported |
|---|---|
| mybatis-plus-geometry | Latest release only |
| geoverse-sar | `main` / latest release only |

## Reporting a Vulnerability

**Please do not report security vulnerabilities through public GitHub issues.**

Instead, use one of these private channels:

1. **GitHub Security Advisories** (preferred): open a
   [private security advisory](https://docs.github.com/en/code-security/security-advisories/guidance-on-reporting-and-writing/privately-reporting-a-security-vulnerability)
   on the affected repository.
2. **Email**: send details to libra.liuyb@gmail.com with `[SECURITY]` in the subject line.

Please include:

- The affected repo and version/commit.
- A description of the vulnerability and its potential impact.
- Steps to reproduce, or a proof-of-concept if available.

We aim to acknowledge reports within a few business days and will work with you on a fix and
disclosure timeline before any public announcement. Please give us reasonable time to address
the issue before disclosing it publicly.

Thank you for helping keep GeoVerse Labs projects and their users safe.
